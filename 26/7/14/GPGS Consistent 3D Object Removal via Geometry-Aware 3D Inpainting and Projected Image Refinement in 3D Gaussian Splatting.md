#### 先几何、后外观

GPGS 的完整流程是：

```
对象分离
→ 参考图像修补
→ 三维几何补全
→ 把参考图像投影到其他视点
→ 修复投影图像的亮度与纹理
→ 用细化后的多视图图像微调 3DGS
```

其中，几何补全负责回答“这个像素在另一个视点应该落在哪里”，外观细化负责回答“落过去以后应该呈现什么亮度和纹理”。两者不是并列关系，而是前后依赖关系。

假设补出的深度偏差很小。投影到相邻视点时，这个偏差可能还不明显；但视点距离越远，误差会被放大，表现为边缘错位、物体轮廓变形、重影或空洞。此时后面的五层 CNN 只能调整二维投影结果，无法知道像素真正应位于哪个三维位置，因此不能可靠修复错误的空间对应关系。

这也是 Figure 7 的关键：一些方法直接渲染时看起来尚可，但把图像依据估计深度重新投影后，结构立即变形。也就是说，“图像看起来合理”不代表“几何能够支持新视点”。完整 GPGS 的几何模块在 Table 3 中取得 SSIM 0.445、LPIPS 0.306、FID 116.822；不做 Point-MAE 场景微调时分别退化到 0.430、0.346、151.462。

亮度迁移和使用 FDL 训练的 CNN 解决的是另一类问题：

- 亮度迁移校正不同视点下的明暗变化；
- CNN 填补投影点不足造成的小范围缺口；
- FDL 在频域比较纹理分布，减轻轻微像素错位对训练的干扰；
- 坐标和图像索引嵌入帮助网络适应视点、位置变化。

所以更准确的表述是：二维细化可以容忍小幅几何误差，但不能代替几何补全。

#### 单参考图像提高一致性

逐视图独立修补的问题不是生成结果“不自然”，而是每张图都可能自然得不一样。例如同一片石板纹理，在视图 A 中被生成成横向裂缝，在视图 B 中可能变成斜向裂缝。单张看都合理，但无法共同解释为同一个三维表面。

GPGS 选择一张参考图像作为内容锚点：

1. 在参考图像中完成一次高质量修补；
2. 用补全后的三维几何建立跨视点对应；
3. 把同一参考内容投影到其他视图；
4. 只对投影结果做亮度和纹理层面的局部调整；
5. 用这些多视图结果重新约束 3DGS。

因此，不同视图中的内容具有共同来源，而不是由多个相互独立的生成过程决定。Figure 6 中，AuraFusion 虽然也接受参考图像引导，但仍依赖扩散模型逐视图生成，输出多样性没有被完全消除；GPGS 的“参考内容投影”约束更强。

代价是系统对参考图像非常敏感。如果参考视图存在以下问题，误差可能被传播到所有视点：

- 未见区域过大，参考图像本身需要大量生成；
- 参考视角不能代表其他方向看到的结构；
- SAM 掩码边界不准确；
- 修补结果包含错误纹理或错误物体结构；
- 强高光、反射等效果无法从单一视图推断。

论文在 COR-NeRF 上手动选择参考图像，在 360-USID 上使用数据集提供的参考图像和未见区域掩码，但没有报告“换一张参考图像后性能变化多少”的敏感性实验。

原文依据：p.2-3 `S007-S008`；p.6 `S017`、Figure 6；实现设置见 p.5 `S015`。

#### Point-MAE 的由粗到细策略究竟在补偿什么

Point-MAE 原本不是为“场景中间有一个巨大空洞”设计的。其基本处理方式是：

- 用 FPS 选取 64 个已有中心点；
- 每个中心点通过 KNN 构造包含 32 个点的局部块；
- 一次预测固定数量的点，论文中为 2,048 个；
- 围绕已有中心点恢复被遮蔽的局部点云块。

问题在于：真正完全空缺的区域没有点，也就没有中心点。Point-MAE 无法凭空在空洞中央启动预测。

GPGS 的由粗到细策略相当于“逐步向空洞内部搭桥”：

1. 粗阶段裁取较大的区域，使空洞只占其中较小比例；
2. Point-MAE 根据周围大量已知点，在空洞边缘预测少量新点；
3. 中间阶段缩小裁块，使空洞占比上升；
4. 把上一阶段预测出的点作为新的中心点；
5. 重复推理，让预测逐渐深入空洞内部；
6. 细阶段通过重叠推理提高点密度。

实现中，作者重复细阶段，直到未见区域的深度覆盖达到 70%，剩余部分再由邻域插值填充。此外，Point-MAE 虽在 ShapeNet 上学到通用三维先验，但作者承认它对未见场景分布可能能力有限，因此还利用当前场景点云和 Chamfer distance 对模型进行场景级微调。

这是一种有效的工程适配，但也带来潜在风险：早期预测错误可能成为后续阶段的新中心点，使误差逐层传播；70% 覆盖阈值、裁块尺度和重叠次数也属于经验设置，论文没有给出这些参数的系统消融。

原文依据：p.3-4 `S009-S010`，Figure 3；具体覆盖阈值见 p.5 `S015`。

#### 感知指标更好，但 PSNR 略低

两个数据集上的对比都呈现相同模式：

| 数据集   | 方法       | PSNR ↑     | SSIM ↑    | LPIPS ↓   | FID ↓       |
| -------- | ---------- | ---------- | --------- | --------- | ----------- |
| COR-NeRF | AuraFusion | **18.832** | 0.510     | 0.294     | 76.003      |
| COR-NeRF | GPGS       | 18.762     | **0.524** | **0.286** | **73.694**  |
| 360-USID | AuraFusion | **18.255** | 0.433     | 0.329     | 146.000     |
| 360-USID | GPGS       | 17.972     | **0.445** | **0.306** | **116.822** |

PSNR 本质上由像素误差决定。若模型无法确定高频纹理，生成一个平滑平均结果有时反而能减少均方误差，因此获得较高 PSNR。清晰但存在少量像素位移的纹理，PSNR 可能更低，却在人眼看来更自然。

LPIPS 更关注深层特征的感知相似性，FID 衡量生成结果与真实图像在特征分布上的距离。GPGS 在这两项上更好，配合 Figure 6 的复杂石板、地毯纹理，支持其结果更清晰、更接近真实外观。

Table 4 还说明了不同组件的作用：

- 去掉 FDL 后，FID 从 116.822 恶化到 147.244，LPIPS 从 0.306 恶化到 0.335；
- 去掉亮度迁移后，PSNR 从 17.972 降到 17.813，SSIM 从 0.445 降到 0.436；
- 完全去掉细化网络后，PSNR 降到 17.637。

不过要谨慎：论文的指标是在对象掩码外接框内按图像计算的，并没有给出专门的跨视图一致性指标，也没有报告方差或统计显著性。因此，“多视图一致性更强”主要由单图指标、定性结果和方法机制共同支持，而不是被一个直接的三维一致性指标完整证明。

原文依据：p.5 `S014`；p.6 `S018`、Table 1-2；p.7 `S020`、Table 4。

#### 实验边界意味着什么

论文明确展示的范围是：

- 两个无界 360° 静态场景数据集；
- COR-NeRF 11 个场景，360-USID 7 个场景；
- 手动选择或由数据集提供单个参考视图；
- 手动借助 SAM 构建未见区域掩码；
- 使用 RaDe-GS 作为底层 3DGS；
- 在单张 RTX 3090 上运行。

因此，论文证明的是“在这类静态、具有可用参考视图和对象掩码的 360° 场景中有效”，还不能直接推出对任意三维场景都有效。

动态场景、透明或强反射对象、非刚体对象、大尺度未见结构，原文均未明确测试。它们分别会破坏静态多视图对应、朗伯外观假设、稳定对象掩码或 Point-MAE 的局部补全前提。特别是透明和反射表面，其外观本来就强烈依赖视点，简单亮度迁移未必足够。

单张 RTX 3090 只说明实验硬件，并不等于已经证明方法轻量。论文没有明确报告完整处理时间、显存峰值、不同场景规模下的增长趋势，也没有与基线比较端到端效率。

更有说服力的后续实验应包括：

- 更换参考视图的敏感性分析；
- 对 SAM 掩码加入边界噪声；
- 按未见区域大小分组评估；
- 使用跨视图重投影误差或三维一致性指标；
- 测试透明、反射、动态和非刚体对象；
- 报告场景级训练时间、显存和点数量扩展性。

原文依据：p.3 `S008`；p.5 `S014-S015`。动态、透明/反射和大尺度空洞的泛化结论属于未覆盖范围，原文未明确说明。
