先纠正一个关键点：我前面阅读提示中提到“TC 模块与 TDL”。这篇论文实际上没有定义名为 Temporal Difference Loss（TDL）的损失项。它使用的是 TC 模块内部的“时序差异特征”，总损失则由传输损失、亮度损失和结构损失组成。把它称为 TDL 不准确。

#### 1. CycleGAN 如何解决非配对训练

临床上很难获得严格配对的数据：同一段肠道、同一视角、同一运动状态下，无法同时拍摄低照度与正常照度视频。因此，SGNet 不学习逐样本的“低照度帧 → 对应正常帧”，而是学习两个数据域之间的映射：

```
低照度域 L ──G1──> 正常照度域 H
正常照度域 H ──G2──> 低照度域 L
```

同时设置两个判别器：

- `D1` 判断 `G1` 生成的正常照度帧是否像真实正常照度帧。
- `D2` 判断 `G2` 生成的低照度帧是否像真实低照度帧。
- `G1 + G2` 构成闭环，要求转换后再转换回来时，仍能恢复原始内容。

这就是 CycleGAN 的“双向域转换”。它解决的不是“生成唯一正确的正常照度图像”，而是“在没有对应真值的情况下，使输出落入正常照度图像的分布，同时尽量保持输入内容”。原文依据：p.5 S046、S052–S068，Fig. 3。

其训练约束包括：

\[ L=L_{\text{trans}}+\alpha L_{\text{ill}}+\beta L_s, \quad \alpha=1,\ \beta=5 \]

其中：

- `Ltrans`：包含对抗损失、循环一致性损失和恒等映射损失。
- `Lcyc`：保证跨域转换后还能还原原始语义内容。
- `Lidt`：输入已经属于目标域时，不应发生不必要的颜色或结构改变。
- `Lill`：约束局部亮度与全局亮度的一致性，避免增强不均匀。
- `Ls`：约束增强前后的结构相似性。

`Ltrans` 内部权重为 `λ1=1、λ2=5、λ3=10`。原文依据：p.8 S189–S226。

需要注意：SGNet 仍然需要低照度视频集合和正常照度视频集合，只是不要求它们内容对应。因此它是“非配对训练”，不是“只用低照度数据训练”。

#### 2. DSCV 数据库为什么是方法的一部分

DSCV 不只是测试集，它同时支撑了 SGNet 的两个基本假设：

- 存在可以学习的低照度域和正常照度域。
- 不同内镜设备具有可描述、可控制的成像风格差异。

数据库包含 300 个视频片段：

- 200 个低照度视频：Fujifilm 100 个、Olympus 100 个。
- 100 个正常照度视频：Fujifilm 50 个、Olympus 50 个。
- 低照度与正常照度视频互不配对。
- 若一个视频中超过 75% 的帧平均灰度低于 100，则划为低照度视频。
- 训练集与测试集按 8:2 划分，并保证内容不重叠。

原文依据：p.4 S039、Table I；p.8 S253。

这意味着论文的贡献实际上是“数据定义 + 无配对映射 + 时序建模 + 设备风格控制”的组合，而不是单独提出一个增强网络。

#### 3. TC 模块如何缓解闪烁

逐帧方法的问题是：每一帧独立决定亮度、颜色和对比度。即使每帧单独看起来不错，相邻帧的增强幅度稍有差异，连续播放时也会出现闪烁、结构抖动或颜色漂移。

SGNet 在训练时采用 `2T+1` 个连续帧，实验中 `T=1`，即每次输入：

```
前一帧 F(t-1) + 目标帧 F(t) + 后一帧 F(t+1)
```

TC 模块大致执行以下过程：

1. 分别提取目标帧和相邻帧的浅层特征。
2. 使用可变形卷积，将相邻帧特征对齐到目标帧。
3. 计算对齐后相邻特征与目标特征的差异。
4. 将前后两个方向的差异特征拼接。
5. 使用通道注意力和空间注意力筛选有用的补偿信息。
6. 经过池化和 ResBlock，形成最终的时间补偿特征。
7. 将补偿特征送入生成器后续编码和解码过程。

原文依据：p.6–7 S093–S155，Fig. 5。

这里“先对齐再求差”非常重要。结肠镜本身在运动，如果直接比较相邻帧，组织位移会被误认为亮度变化。可变形卷积先处理运动造成的位置偏差，之后提取的差异才更接近真正需要补偿的亮度、纹理和曝光变化。

但论文没有使用一个显式的“相邻输出帧必须相似”的 TDL。它主要依靠：

- 多帧联合输入；
- 特征对齐；
- 时序差异特征；
- 注意力筛选；
- 相邻帧信息补偿；

把时序一致性嵌入网络结构。

消融实验支持 TC 有效：移除 TC 后，网络退化为逐帧处理，`WaDIQaM-NR` 和 `DARNet-B` 分别下降约 17.57% 和 5.68%；Fig. 9 也显示加入 TC 后纹理更完整、亮度过渡更平滑。原文依据：p.10 S271，Table VI，Fig. 9。

#### 4. SG 模块如何控制设备风格

论文认为“正常照度”并不是唯一风格。不同设备即使拍摄正常视频，也存在系统性差异：

- Fujifilm：偏冷色调、较低饱和度、中等对比度。
- Olympus：偏暖色调、较高饱和度、较高对比度。

这些差异通过 CIE Lab 色调、HSV 饱和度和局部对比度进行了统计。原文依据：p.4 S039–S044，Fig. 1、Fig. 2、Table II。

SG 模块的处理链是：

```
设备风格属性
→ 文本描述
→ BGE-Micro-v2 文本编码
→ 风格特征 Ftext
→ 与视频特征进行交叉注意力
→ 风格引导的视频特征 Fstyle
→ 解码生成目标风格视频
```

作者先把设备特征概括成短语，再用 GPT-4 生成五种语义一致但表达不同的描述，随后由资深医生审核。文本通过冻结参数的 BGE-Micro-v2 编码。原文依据：p.7 S157–S159，Table III。

在 SG 模块内部：

- 文本特征作为 Query。
- 视频特征作为 Key 和 Value。
- 视频特征先加入基于深度卷积的位置编码。
- 多头交叉注意力计算“哪些视觉位置和通道与目标风格最相关”。
- 注意力结果经过 Sigmoid 门控后作用于视频特征。
- 最后加入残差连接，尽量保留原始内容。

原文依据：p.7–8 S160–S187，Fig. 6。

因此，文本不是直接告诉网络“把图像调亮多少”，而是提供一个风格条件，控制色调、饱和度和对比度等属性如何保留或迁移。

文字消融进一步表明：

- Neutral Text：只说这是内镜图像，不包含具体风格。
- Single-Sentence Text：每种设备使用一条风格描述。
- Multi-Sentence Text：使用设备描述和五种扩展表达。

加入具体风格信息优于 Neutral Text，多描述又优于单描述。原文依据：p.10–11 S273、S278–S279，Table VII。

不过，“灵活风格迁移”目前主要在 Fujifilm 和 Olympus 两类设备上验证，不能直接等同于对任意新设备或任意自然语言风格都能零样本泛化。

#### 5. 为什么必须同时看 IQA 和 VQA

论文采用两组无参考指标。

单帧图像质量评价（IQA）：

| 指标       | 主要关注             | 趋势     |
| ---------- | -------------------- | -------- |
| BRISQUE    | 自然图像统计失真     | 越低越好 |
| DBCNN      | 学习到的图像质量特征 | 越高越好 |
| WaDIQaM-NR | 局部结构和感知质量   | 越高越好 |

视频质量评价（VQA）：

| 指标     | 主要关注                 | 趋势     |
| -------- | ------------------------ | -------- |
| DOVER    | 自然视频的技术与审美质量 | 越高越好 |
| DARNet   | 结肠镜视频整体质量       | 越高越好 |
| DARNet-B | 结肠镜视频亮度质量       | 越高越好 |

原文依据：p.8–9 S254–S255。

IQA 是把每帧分别评分后取平均值。它看不到一个典型问题：假设每帧都很清晰，但亮度在相邻帧之间反复跳动，平均单帧分数仍可能很高，播放时却会严重闪烁。

VQA 则把时空变化考虑进去，更适合检验 TC 的目标。因此，只有同时改善 IQA 和 VQA，才能说明模型不是简单地“把每帧调亮”，而是在增强纹理与自然度的同时改善视频连续性。

SGNet 在 Table IV 中得到：

- BRISQUE：27.0311
- DBCNN：0.5002
- WaDIQaM-NR：−0.2969
- DOVER：0.5689
- DARNet：4.0979
- DARNet-B：2.2939

相较 StillGAN，SGNet 的 WaDIQaM-NR、DOVER 和 DARNet 分别提高 18.46%、11.66% 和 7.09%。作者据此把优势归因于相邻帧时序相关性的利用。原文依据：p.9 S259–S262，Table IV。

#### 6. 应当如何审慎理解结果

这篇论文证明得比较充分的是：

- 不需要内容严格配对的训练视频。
- TC 和 SG 都对无参考质量指标有贡献。
- 在 DSCV 的两种设备上，结果优于六种逐帧或无配对对比方法。
- 输出在视觉上更自然、连续，并较好保持设备风格。

尚未充分证明的是：

- 是否提高息肉检出率或诊断准确率；
- 是否在更多医院、设备和患者分布上泛化；
- 是否优于专门的视频增强监督方法；
- 无参考指标的提升是否完全等同于临床可用性。

此外，SGNet 在 `256×256` 输入下需要 57.745 GFLOPs、20.585M 参数和约 11.25 ms 推理时间，明显重于 SCI、SGZ 等轻量方法。作者也将轻量化和更多设备验证列为后续工作。原文依据：p.10 S266–S269，Table V；p.11 S282。
