### [A Survey on 3D Gaussian Splatting](https://arxiv.org/pdf/2401.03890)

这四条实际上构成了一条完整的论文阅读链：

```
理解 3D GS 为什么快 → 找出它在哪些地方失效 → 看这些能力如何支撑应用 → 从尚未解决的矛盾中寻找研究问题
```

#### 1. 第 3 节：3D GS 为什么能够实时渲染

第 3 节回答两个核心问题：

1. 给定一组 3D 高斯，怎样快速渲染图像？
2. 怎样从训练图像中学到数量和属性都合适的 3D 高斯？

##### 1.1 场景表示

一个 3D 高斯基元主要包含：

- 中心位置 $(\mu)$
- 不透明度 $(\alpha)$
- 三维协方差矩阵 $(\Sigma)$，决定高斯的大小、方向和形状
- 由球谐函数 SH 表示的颜色，可表达视角相关外观

因此，它不只是一个彩色点，而是一个具有空间范围、方向、透明度和视角相关颜色的小椭球体。场景由数百万个这样的显式基元组成。因为属性直接存储，渲染时不必像 NeRF 那样为每条光线反复查询 MLP。原文依据：p.5 S024-S025。

##### 1.2 Fig. 2 应该怎样读

Fig. 2 展示的是以下流水线：

```
3D 高斯 → 视锥剔除 → 投影到图像平面 → 分配到图块 → 按图块和深度排序 → Alpha 合成
```

###### 第一步：视锥剔除

先排除相机视野之外的高斯。这样后续投影、排序和颜色计算只处理可能出现在当前图像中的基元。原文依据：p.6 S026。

###### 第二步：Splatting 投影

Fig. 2a 中，空间里的 3D 椭球被投影成图像平面上的 2D 椭圆。

投影不只是把高斯中心从三维变成二维，还必须把三维协方差变成二维协方差：

$[ \Sigma' = JW\Sigma W^\top J^\top ]$

其中 \(W\) 是观察变换，\(J\) 是投影变换局部仿射近似的雅可比矩阵。二维协方差决定了该高斯在屏幕上的覆盖范围、方向和形状。原文依据：p.6 S027-S029，Eq. 4；Fig. 2：p.6 F002/C002。

###### 第三步：确定重叠高斯并按深度排序

对某个像素而言，可能有多个投影后的高斯覆盖它。必须按相机深度从前向后排列，否则透明度混合顺序错误时，颜色和遮挡关系都会出错。

最直接的方法是为每个像素分别寻找、排序其覆盖高斯，但这样代价很高，而且难以充分并行。原文依据：p.6 S030-S034。

###### 第四步：Alpha 合成

排序后，像素颜色采用前向后 Alpha 合成：

$[ C=\sum_{n=1}^{|\mathcal N|} c_n\alpha'_n \prod_{j=1}^{n-1}(1-\alpha'_j) ]$

直观上：

- $(c_n\alpha'_n)$：第 \(n\) 个高斯自身贡献的颜色；
- $(\prod_{j<n}(1-\alpha'_j))$：前面所有高斯之后还剩多少透射率；
- 越靠前、越不透明的高斯，对最终颜色影响越大；
- 后方高斯的贡献会被前方高斯逐渐遮挡。

这里的 $(\alpha'_n) $又由高斯自身不透明度与像素到二维高斯中心的距离共同决定。像素越靠近高斯中心，贡献通常越大。原文依据：p.6 S030-S033，Eq. 5-6。

##### 1.3 为什么需要图块

逐像素建立高斯列表和排序太慢。3D GS 因此牺牲一部分粒度，把图像划分为通常为 $(16\times16)$ 像素的图块 tile。

对于每个投影高斯：

- 计算它覆盖哪些图块；
- 如果覆盖多个图块，就为每个图块复制一份记录；
- 每份记录附上图块 ID 和深度；
- 先按图块 ID、再按深度排序。

这样，同一图块中的像素共享同一份有序高斯列表，不再为每个像素单独排序。原文依据：p.7 S035。

##### 1.4 GPU 并行来自哪里

排序键把图块 ID 放在高位、深度放在低位。排序完成后：

- 同一图块的高斯天然连续存放；
- 不同图块可并行处理；
- 同一图块中的像素也可并行；
- 像素从共享内存读取相同的高斯序列；
- CUDA 中通常把图块映射为 block、像素映射为 thread。

这才是 3D GS 实时渲染的关键。它的速度不只是因为“高斯比神经网络简单”，而是显式表示、图块化、排序编码、共享内存和 GPU 并行共同产生的结果。原文依据：p.7 S036；Fig. S3：p.38 F006/C006。

##### 1.5 渲染之外，还要读懂优化

第 3.2 节说明高斯是怎样学出来的。

参数优化主要包括：

- 使用 $(L_1)$ 保证像素颜色接近；
- 使用 D-SSIM 保留结构、高频细节和局部亮度模式；
- 不直接优化任意协方差矩阵，而是优化旋转四元数 $(q)$ 和尺度向量 $(s)$，从而保证协方差具有合法结构。

原文依据：p.7-8 S037-S043，Eq. 7-8。

密度控制则决定“哪里需要多少个高斯”：

- 初始化：来自 SfM 稀疏点云或随机点；
- 克隆：重建不足且高斯较小时，复制并沿梯度方向移动；
- 分裂：高斯太大时，用两个更小的高斯替代；
- 剪枝：删除近乎透明、过大或贡献较低的高斯；
- 不透明度重置：防止相机附近高斯数量失控。

原文依据：p.8 S044-S046。

所以，第 3 节真正需要掌握的是两条回路：

- 前向回路：高斯 → 投影 → 排序 → 合成 → 图像；
- 训练回路：图像误差 → 参数更新 → 致密化/剪枝 → 更合理的高斯集合。

#### 2. 第 4 节：从能力瓶颈选择方法

第 4 节不是简单按论文年份罗列工作，而是按照“原始 3D GS 在哪里不够用”组织研究方向。Table S1 是全局索引，Table S2 则专门比较稀疏输入方法。原文依据：p.8 S047；Table S1：p.38 T001/C007。

| 瓶颈       | 主要思路                                    | 收益                             | 代价或风险                         |
| ---------- | ------------------------------------------- | -------------------------------- | ---------------------------------- |
| 稀疏输入   | 深度等正则化；生成先验；前馈高斯预测        | 少视图甚至单图重建               | 几何与速度难兼得，先验可能幻觉     |
| 内存       | 减少高斯数量；属性量化、码本和压缩          | 降低存储和显存                   | 高压缩率下质量下降，训练显存仍高   |
| 真实感     | 抗锯齿、反射、透明、模糊和重光照            | 提升视觉质量                     | 通常只解决特定伪影，管线更复杂     |
| 优化       | 几何/频率正则化；改进致密化；无 COLMAP 优化 | 更快收敛、更少漂浮物             | 超参数多，通常仍需逐场景优化       |
| 更多属性   | 加入语言、语义、时间和物理属性              | 开放词汇查询、动态建模           | 内存和监督需求增加                 |
| 混合表示   | 结合 MLP、网格、形变场或参数化人体模型      | 更强结构和可控性                 | 牺牲纯 GS 的简单性与通用性         |
| 新渲染算法 | 用光线追踪或物理渲染替代单纯光栅化          | 更好处理反射、二次光线和复杂相机 | 计算成本高，实时集成尚不成熟       |
| 核函数工程 | 改变高斯形状、支撑域或泼溅算子              | 更好表示锐利边界和薄表面         | 需要定制渲染器，兼容性和稳定性下降 |

##### 2.1 Table S2 的核心不是“谁更好”，而是两种范式的交换

稀疏输入方法被分成两类。

| 维度       | 基于正则化             | 基于泛化性                       |
| ---------- | ---------------------- | -------------------------------- |
| 推理方式   | 每个场景单独优化，较慢 | 前馈预测，较快                   |
| 几何质量   | 通常细节和几何更好     | 可能精度较低或产生先验幻觉       |
| 测试输入   | 通常仍需要多个视图     | 可使用少视图甚至单图             |
| 未观测区域 | 容易过拟合已观测部分   | 可补全，但结果可能“合理却不准确” |
| 典型用途   | 高保真离线重建         | 实时或交互式新视角合成           |

原文依据：p.9 S048-S050；Table S2：p.38 T002/C008。

最值得注意的是，作者没有认为两类方法必须二选一，而是提出了自适应控制这种权衡的可能性，例如：

- 学习置信度；
- 根据上下文选择先验；
- 根据用户对速度或质量的偏好切换策略。

这其实是一个很直接的研究切入点：让系统判断“当前区域应该相信观测数据，还是相信学习先验”。原文依据：p.9 S050。

##### 2.2 实际选方法时先回答五个问题

1. 输入是多视图、少视图还是单图？
2. 是否允许逐场景训练，还是必须一次前馈完成？
3. 主要失败模式是锯齿、模糊、反射、几何错误还是内存不足？
4. 是否需要语义、动态、物理或编辑能力？
5. 部署约束是画质优先、FPS 优先、显存优先还是可控性优先？

第 4 节的价值，就是让你从这些约束出发选方法，而不是看排行榜最高分就使用某个模型。

#### 3. 第 5-6 节：应用需求如何对应性能证据

第 5 节回答“3D GS 能用在哪里”，第 6 节回答“在若干代表性任务上，证据有多强”。

| 应用        | 3D GS 的优势                         | 主要瓶颈                                 | 对应证据                    |
| ----------- | ------------------------------------ | ---------------------------------------- | --------------------------- |
| 机器人/SLAM | 实时稠密建图、显式几何、便于操作     | 动态环境、漂移、多传感器标定、语义稳定性 | p.13-14 S071-S073；Table S4 |
| 动态场景    | 可为高斯增加时间、形变或 4D 属性     | 长序列、对象级运动、动态/静态解耦        | p.14-15 S074-S076；Table S6 |
| 生成与编辑  | 可微渲染，基元可直接移动、删除或修改 | 多视图一致性、Janus 问题、计算成本       | p.15-16 S077-S079           |
| Avatar      | 高速渲染，易与人体模型或网格绑定     | 遮挡、宽松衣物、少视图和交互性           | p.16-17 S080-S082；Table S7 |
| 内窥镜      | 高速、显存需求较低，适合动态软组织   | 单视图、工具遮挡、非刚性形变             | p.17 S083-S085；Table S8    |
| 大规模场景  | 分块训练和 LoD 可扩展到城市/航拍     | 稀疏采集、跨块一致性、存储和加载         | p.18 S086-S088              |
| 物理模拟    | 高斯可同时作为渲染基元和模拟粒子     | 物理监督弱、材料类型有限、长期误差累积   | p.18-19 S089-S091           |

##### 3.1 第 6 节的比较逻辑

每一个任务基本都按照以下顺序组织：

```
数据集 → 纳入的算法 → 评价指标 → 结果
```

例如：

- 定位：Replica，指标是 ATE，而不是 PSNR；p.19 S094-S098，Table S4。
- 静态建图：Replica，指标是 PSNR、SSIM、LPIPS 和 FPS；p.20 S099-S103，Table S5。
- 动态场景：D-NeRF，比较动态渲染质量；p.20 S104-S108，Table S6。
- Avatar：ZJU-MoCap；p.20-21 S109-S113，Table S7。
- 手术场景：EndoNeRF，同时看画质、FPS 和显存；p.21 S114-S118，Table S8。

##### 3.2 为什么不能只按 PSNR 排名

PSNR 只衡量像素误差，它不等价于：

- 几何是否准确；
- 相机轨迹是否准确；
- 动态运动是否合理；
- 反射或透明物体是否真实；
- 是否能实时运行；
- 是否支持编辑；
- 是否适合目标硬件。

因此，完整比较至少应包括：

- 图像质量：PSNR、SSIM、LPIPS；
- 几何或定位：深度误差、ATE、表面误差；
- 效率：训练时间、FPS、延迟；
- 资源：峰值显存、模型大小；
- 稳定性：不同场景和初始化下的方差；
- 功能：动态、编辑、语义、物理和可控性。

##### 3.3 即使同一数据集，也不能直接抄表排序

作者明确提醒：

- 致密化计划、剪枝和不透明度重置会显著影响结果；
- 不同方法可能使用不同实现、超参数和预处理；
- FPS 依赖 GPU、分辨率和代码优化；
- 显存和存储随场景尺度、纹理、视图数量和目标质量变化；
- 很多分数取自各自原论文，而不是在统一代码库中重新训练。

原文依据：p.19 S092-S093。

所以更可靠的比较顺序是：

1. 先确认数据集、训练/测试划分和输入条件相同；
2. 再确认指标定义和硬件相同；
3. 然后比较质量-速度-显存的 Pareto 前沿；
4. 最后检查该方法是否解决你的目标失败模式。

#### 4. 第 7 节：怎样从开放问题找到研究切入点

“数据、物理真实性、鲁棒性、效率与统一表示”是对第 7 节及前文瓶颈的归纳，并不是作者逐字给出的五个小标题。

##### 4.1 数据与泛化

趋势正在从“每个场景优化一次”转向“前馈预测高斯”。新的模型利用 U-Net、Transformer、混合注意力或测试时训练处理越来越大的多视图上下文，并尝试同时预测相机、几何和动态信息。原文依据：p.21 S119。

可研究的问题包括：

- 少视图下怎样避免先验幻觉？
- 未标定图像能否同时恢复相机和高斯？
- 怎样建立跨对象、跨场景的高斯先验？
- 测试时适配能否兼顾速度和几何精度？

##### 4.2 物理真实性

作者认为，高斯不仅可以表示外观，还可以承载语义、材料和动力学信息。但当前系统往往只支持有限的弹性或材料类型，尚不能统一处理刚体、弹性体、塑性体、颗粒和流体。原文依据：p.22 S120-S121；p.18-19 S089-S091。

可研究的问题包括：

- 语义标签能否自动决定材料模型？
- 能否从视频学习质量、刚度、摩擦等物理参数？
- 怎样让扩散模型提供的“视觉真实”转化为定量物理正确？
- 如何结合 real2sim 与 sim2real 学习可泛化动力学？

##### 4.3 鲁棒性

鲁棒性问题散布在多种应用中：

- SLAM 面临动态环境、漂移和传感器退化；p.14 S073。
- 动态重建面临长序列和对象级运动推理；p.15 S076。
- 内窥镜面临单视图、遮挡和非刚性形变；p.17 S084-S085。
- 反射、透明、模糊和低纹理场景仍不稳定；p.10 S054-S056。

好的研究问题不应只是“再提高一点平均 PSNR”，而应明确：

- 在什么退化条件下失败？
- 失败是几何、外观、相机还是先验造成的？
- 能否估计不确定性并自动切换策略？
- 最坏情况性能是否得到改善？

##### 4.4 效率

效率包括四个不同层面：

- 渲染效率：图块化、排序和 CUDA 核函数；
- 优化效率：减少逐场景训练和超参数敏感性；
- 内存效率：减少高斯数量、量化属性和构建码本；
- 大场景效率：分块训练、LoD、加载和流式传输。

原文依据：p.9 S051-S053；p.12-13 S067-S069；p.18 S086-S088；p.21 S119。

因此“更快”必须具体说明是：

- 更高 FPS；
- 更短训练时间；
- 更低峰值显存；
- 更小模型；
- 还是更低端到端延迟。

##### 4.5 统一表示

最终目标可能是让同一组高斯同时支持：

- 渲染与几何；
- 语义查询；
- 动态和时间；
- 物理模拟；
- 编辑与控制；
- 机器人感知和行动。

但每增加一种属性或结构，都会损失纯 3D GS 的简洁性，并增加显存、监督和计算成本。原文依据：p.11-13 S060-S069；p.21-23 S119-S124。

特别值得注意的开放问题是内部结构：当前高斯通常是无组织的表面/外观基元，很难忠实表示 CT 一类真正的体积内部结构。作者明确认为这一问题尚无成熟答案。原文依据：p.22 S122。

#### 可以直接采用的选题框架

一个有说服力的选题最好同时明确四件事：

> 在特定失败条件下，通过一种明确机制，改善一组可测指标，同时满足实际资源约束。

例如：

- 稀疏动态输入下，用不确定性感知策略在正则化与生成先验之间切换；
- 将语义类别映射到材料模型，构建物理-语义统一高斯；
- 为 CT 或工业成像设计具有拓扑/体积约束的高斯表示；
- 面向城市仿真联合优化 LoD、动态对象、重光照和物理运动；
- 将前馈 Transformer 与高斯码本压缩结合，研究质量-延迟-显存的 Pareto 前沿。

这些是基于原文开放问题形成的研究建议，不是作者已经完成的工作。
