### [2403.02693](https://arxiv.org/pdf/2403.02693)

#### Critical Reading Notes

1. 本文的核心贡献不是单一模型结构，而是把显著性预测、用户历史视口建模、移动端开销控制和码率自适应放在同一个系统约束下协同设计。
2. 需要重点检查实验公平性：服务器端显著性预测和客户端推理成本被分开计算，比较方法是否具备同等系统优化空间会影响结论解释。
3. MAML 快速适配是亮点，但实际直播部署还取决于新视频少样本采集延迟、服务器 GPU 资源和显著性图传输频率。
4. QoE 改善主要来自更准的视口预测和 CBA 对概率输出的利用；Fig. 17 是连接预测精度与用户体验的关键证据。

#### 移动端

使用作者修改后的 `ConvLSTM` 视口预测模型，而不是 GCN，也不是 MAML。

它的输入包括：

- 服务器传来的显著性图
- 当前用户过去的视口轨迹，转换为 tile 矩阵

输出是未来每个 tile 出现在用户视口中的概率。

为实现轻量部署，作者进行了三项处理：

1. 把 ConvLSTM 中的普通卷积替换为深度可分离卷积 `Depthwise Separable Convolution`，降低参数量和计算量。

2. 加入轻量级 `SE block`，重新加权显著性图和历史视口等不同通道。

3. 不直接使用 Keras 的 ConvLSTM 接口，而是用 TensorFlow Lite 支持的基础算子重新实现 Modified ConvLSTMCell。

   | 模型                     | FLOPs | 模型大小  | Mate 30 推理时间 |
   | ------------------------ | ----- | --------- | ---------------- |
   | LR                       | <1M   | <1 KB     | <1 ms            |
   | MobileNetV2，文中记为 MN | 1401M | 13,790 KB | 483 ms           |
   | Modified ConvLSTM        | 149M  | 710 KB    | 330 ms           |

因此它比 MobileNetV2 约快 `1.4×`、小 `19×`，同时准确率更高。虽然 LR 极快，但只能应付较短预测窗口，准确率明显较低。

完整的深度可分离卷积 + SE 配置达到：

- Accuracy：`93.71%`
- F1-score：`86.60%`
- 推理时间：`330 ms`

SE-only 配置准确率略高，为 `93.81%`，但耗时达到 `372 ms`；最终方案选的是更均衡的组合。
模型还采用 float16 量化，论文报告模型缩减到 `710 KB`；ZIP 压缩后传输文件约 `10 KB`，而且只需向客户端发送一次。

#### 服务器端

服务器端包含两个模块：

- MAML-based Few-shot Fast Trainer
- 基于 SalGCN 的显著性预测网络

SalGCN 是面向 360° 球面图像的 GCN：

- 将 ERP 帧转换为球面图结构
- 使用类似 U-Net 的 encoder-decoder
- 使用 ChebNet 图卷积层
- 输出 ERP 格式显著性图

使用球面 GCN 是为了避免普通二维 CNN 处理 ERP 投影时产生的畸变问题。依据见 p.4–5 `S004–S005`、Fig. 2。

实验中的服务器硬件是 Nvidia Tesla V100 GPU。论文报告服务器执行一次显著性预测约需 `0.65 s`。显著性图下采样后按每个视频 chunk 一张发送给客户端，经验下采样比例设为 `144`。

#### MAML 的好处

MAML 适配的不是移动端 ConvLSTM，而是服务器端针对新视频的 SalGCN 显著性模型。这一点很关键。

直播内容无法提前知道，普通预训练模型切换到新视频、场景或内容分布后，性能可能迅速下降。如果为每个新视频从头训练 SalGCN，又无法满足直播实时性。

MAML 学习的不是一个固定视频模型，而是一个“容易快速适配”的参数初始化：

1. 把不同视频的显著性预测看成不同任务。
2. 每个任务划分 support set 和 query set。
3. 内循环用少量 support 样本更新模型。
4. 外循环根据 query loss 更新初始化参数。
5. 新直播到来时，从这个初始化出发，只需少量梯度更新即可得到当前视频的 SalGCN。

实验中，每个新任务只使用 `5` 个 support samples，更新 `10 epochs`。MAML 在第一次梯度更新后性能就快速提高，后续持续优于普通预训练模型。

所以论文真正的亮点可以概括为：

> 用服务器端 MAML + SalGCN 保留复杂模型的内容理解能力，用移动端轻量 Modified ConvLSTM 完成个性化实时预测，再通过下采样、量化和低频传输控制两端通信成本。

MAML 解决的是“新视频内容不断变化，复杂服务器模型如何快速适配”；Modified ConvLSTM 解决的是“移动端如何实时、低成本地融合内容显著性和个人行为”。两者承担的是不同问题。
